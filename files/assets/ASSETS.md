## Assets Outline

Use this file to track all project assets in one place.

### Asset Categories

1. Images
- Figures for report
- Screenshots of implementation
- UI or prototype captures

2. Diagrams
- Architecture diagrams
- Workflow/process diagrams
- Threat or defense chain diagrams

3. Presentation Files
- Slide deck source (PPTX/Keynote)
- Exported presentation (PDF)
- Speaker notes (optional)

4. Data Visuals
- Charts used in report
- Graph exports (PNG/SVG)
- Tables as image or CSV support

5. Media (Optional)
- Demo videos
- Audio narration
- GIF walkthroughs

6. Branding and Layout (Optional)
- Logo files
- Cover page visuals
- Icons and visual style resources

### Naming Convention

- Use lowercase with hyphens.
- Include type when useful.
- Keep names short and descriptive.

Examples:
- `system-architecture-v1.png`
- `threat-chain-final.svg`
- `final-presentation-v1.pdf`

### Organization Suggestion

```text
files/assets/
├── images/
├── diagrams/
├── slides/
├── visuals/
└── media/
```

### Quality Checklist

- Asset has clear source attribution
- License/usage is verified
- File name follows convention
- Referenced in report or slides
- Final version marked and archived
